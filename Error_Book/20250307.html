    所用材料为CrFeCoNiPd高熵合金。

    # 1. 应力应变曲线图分析

    # 1.1 ISPIN=2的应力应变曲线图（小变形）

    问题1: 使用的初始构型是在ISPIN=2下进行结构优化的CrFeCoNiPd高熵合金构型，在结构优化的最后一步中，计算的x方向的应力为-0.065783GPa。但是在拟合应力应变曲线时，我没有代入这个结果（100个数据点是从-0.05到0.05的应变，其中不包含“平衡位置”）。但是图中的结果在应变为0的附近，显然没有得到接近0的应力结果。

    问题2：相比于下面将参数调为ISPIN=1，应力应变曲线的噪声较大，难以获取弹性常数。

    # 1.2 ISPIN=1的应力应变曲线图（小变形）

    在问题1，即0应变处不平衡的问题更加明显，但是这一现象我暂时认为是合理的，因为初始构型是在打开磁性计算时得到的（即ISPIN=2）

    在问题2方面，噪声现象得到了极大的改善，可以看到在应力为0附近的线性段。

    # 1.4 再次绘制ISPIN=1的应力应变曲线图（小变形）

    在0应变的应力仍然不为0,我对比了结构优化和测量应变的INCAR脚本，除了将NSW调整为1意外事件，没有任何差别。(我认为，原因可能是用ASE从XDATCAR到POSCAR的格式转化中的精度损失，在附录中对比了两者前几行的差别)

    通过分析拟合应变为0附近的点，估算弹性模量的xx分量约为247GPa,xy的分量约为205GPa。xx方向为[100]方向，yy方向为[010]方向。

    在Li Teng的工作，没有直接给出弹性常数，但是绘制了一幅压缩的应力应变曲线图，根据曲线图估计，弹性模量约为210GPa，并且根据观察应该也是100晶面。

    # 1.5 小结

    ISPIN=1和ISPIN=2的计算结果差距较大。ISPIN=2的结果有明显的噪声，并且ISPIN=2的收敛率大约为1/50（最近计算了600+样本，有大约15个构型不收敛）。

    弹性模量方面和Li Teng的工作也存在比较大的差距。但是因为我是肉眼估计他的结果，感觉也在可以接受的范围，至少量级相当。

    并且初始应力不为0的问题还是有些奇怪。

    # 3. 对于Cauchy-Born rule的调研

    # 3.1 简介

    温度范围：0K，绝对零度；

    力学范围：弹性形变；

    Rule定位：atomsitic models 和 continuum models之间的一个桥梁。

    发展历史：1950年，Cauchy从原子间的对势推导出了线弹性模量的表达式，并得到了著名的Cauchy关系。后来被Born和Huang进一步发展得到Cauchy-Born rule，再后来有许多科学研究围绕其为主题进行理论和实验研究。

    # 3.2 初步理解

    宏观力学上的弹性变形得到的Cauchy应变张量可以用于在原子尺度分析对应晶体构型形变。

    # 6. 交流总结

    # 6.1 注意事项

    1. 测量弹性模量时，一般是分析千分之一，万分之一的部分，到0.05的应变已经算是大过头了；

    2. 如果是结构优化的最终构形，其实不用从XDATCAR中提取，可以直接用输出中的CONTCAR文件，可以有效的避免ASE格式转化的精度问题；

    3. 理论上打开ISPIN=2是算得更准的，噪声可能和Cauchy-Born rule有关，即对于合金构型，其需要使用complex structure的理论，所以不同元素不会一起扩缩。

    # 附录2 单轴拉伸脚本

    #!/bin/bash

mkdir tmp
cp POSCAR POSCAR-01
mv POSCAR-01 ./tmp/

for i in $(seq 1 1 100);
do
    #le:less than or equal
    if [ $i -le 50 ]; then
            ii=$(echo "scale=4; $i/1000" | bc)
    else
            tmp=$(($i - 50))
            ii=$(echo "scale=4; $tmp/-1000" | bc)
    fi

    cd tmp/
    atomsk POSCAR-01 -sort species up -frac -deform x $ii POSCAR
    mv POSCAR ../POSCAR-$(printf "%04d" $i)
    cd ..
done

rm -r tmp/
