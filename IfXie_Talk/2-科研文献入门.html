      # P1 科研文献入门

      前3部分主要介绍文献是什么，第4部分简单地介绍文献的管理，第5部分介绍文献阅读，第6部分介绍文献查找；

      针对内容是理工科科研文献，所以自然没有对于中文文献的讨论。

      # 1. 文献类型（按照内容分类）：

      主要分为3类，分别是综述（Review），快报（letter），研究文章（article,paper）

      # 1.1 综述

      综述主要内容是对于某一研究领域内，科研成果的总结概括；

      好的综述一般为约稿，即期刊出钱请领域内的大佬写综述，并刊登在自家期刊上。

      # 1.2 快报

      比较简短，多为科研的初步成果；

      在部分学校，可能认为“快报”算为一种“科研文章”，但是一般快报的地位都是低于同级别科研文章的。

      # 1.3 研究文章

      最为常见的论文，包含研究背景，研究方法，研究结果；

      # 2. 科学引文索引（SCI, Science Citation Index）

      逻辑顺序是，某篇文献是属于某一期刊的，而受到认可的期刊就会被归类到SCI之中。

      # 3. 期刊分级

      注：在第一部分是对于文献分类，这里是对于期刊分类；

      评价期刊主要通过两个方面，一方面是影响因子（IF, Impact factor），另一方面是文章分区（而分级有两种主流分区）。

      # 3.1 影响因子

      影响因子是和期刊有关的，并且是存在一定波动的。因为影响因子是与期刊中所有文献在特定年份被引用的频率相关。

      （所以如果今年的某篇期刊引用率下降了，这篇期刊的影响因子就会下降）

      一般影响因子和研究领域也是强相关的，如果一个领域比较热门，其文章引用率自然比较高。

      # 3.2 JCR分区

      JCR分区是分了4个区，分别是Q1,Q2,Q3,Q4,每个区的占比都是1/4

      # 3.3 中科院分区

      中科院分区也是分了4个区，分别是一区，二区，三区，四区，但是占比不同，一区的占比是0.05，二区的占比是0.2，三区的占比是0.25，四区的占比是0.5

      # 4. 文献管理

      从师兄师姐的经验可以得知，在科研期间阅读文献的数量是极大的，所以在入门是有意识地培养文献管理经验是比较重要的。

      # 4.1 文献管理软件

      选择文献管理软件最好的方式就是问导师使用的是哪一款软件；（如果一开始不好意思问导师，也可以问师兄师姐，但是师兄师姐可能和导师用不同的文献管理软件）

      比如某课题组，导师使用的是EndNote，师兄师姐使用的是Zotero

      使用Zotero的师兄师姐认为，Zotero的优势在于阅读文献的便捷性（比如一键翻译，笔记注释等）

      而我认为EndNote在抛开价格以外（北大有免费正版软件），其优势就是良好的生态环境（尤其在于文献导入，文献信息记录方面，文献导出，甚至直接导出一整个数据库）而我确实也感觉它自带的pdf阅读有点丑，还不如用pdf阅读器打开。

      # 4.2 经验总结

      总而言之，我自己的习惯用EndNote进行文献收集与管理，然后打印为纸质版阅读/导出pdf在iPad上阅读（毕竟手动做笔记才是最爽的）而使用zotero就是收集，阅读，整理一体化。

      其实我自己一开始是使用免费开源的Zotero过来的（也因为励老师用的是Zotero），后来是在张老师的建议下进行了更改。所以在科研最初入门时，可以先两个都进行尝试（其实两个软件很多地方有相似性），最重要的是培养文献管理的意识。

      # 5. 文献阅读

      我们在第一部分已经介绍了按照内容可以对于文献进行分类，所以不同分类的文献需要使用不同的阅读方法，以及有着不同的阅读目的

      # 5.1 综述

      励老师让我读的第一篇文献就是一篇曹丕忠老师对于无损检测领域的综述文献，这种文献的特点是篇幅长，内容多。

      根据励老师的建议，好的综述文献是需要反复阅读的，这里先介绍综述文献的初读：

      1. 记录研究领域内的需要被解决的问题；（从大领域中抽出小课题）

      2. 记录解决这些问题的方法；（这一步不需要彻彻底底地掌握，只需要知道这些方法是什么）

      3. 记录哪些方法可以解决哪些问题，在解决过程中有哪些优势和局限。（这是需要思考的地方，初读中最为重要的地方）

      # 5.2 研究文章（快报也相似）

      研究文章也分为泛读和精读，这里先介绍泛读：

      1. 先阅读摘要，总结出3个点：（1）“这篇文章解决了什么问题”（2）“这篇文章用了什么方法解决了这个问题”（3）“这个方法在解决这个问题的优势和亮点在哪”

      2. 然后阅读研究背景介绍，这里可以进一步地确定综述的3个点，以及大概会补充一些其他研究方法，然后进行优势比较。

      3. 对于研究方法，不用太细的阅读；（如果不复现的话）

      4. 对于实验和结果部分（这里的实验也包括coding中参数设置）是需要认真读的，因为在这一部分很容易收获一些经验，比如别人的训练集怎么设置的，别人实验参数是怎么选取的等。（这部分也很容易判断出文章的优劣，有些不好的文章在这一部分有明显错误）

      5. 最后总结部分一般都是方法总结，可能有提及局限，大多为未来展望。

      # 5.3 为什么阅读文献

      1. 读文献就是在写文献，作为研究生多看看自己目标期刊的文献，在自己动笔时也有可以参考的思路；（参考的只是行文思路，不是抄袭内容）

      2. 形成自己的学术品味，首先是了解这个大领域中的各个子问题，了解解决问题的各个方法的特点，然后思考哪些问题是自己可以做，为什么做，该怎么做等。渐渐地也可以和拥有相近学术品味的人产生共鸣，增强理解和联系感。

      # 6. 文献查找

      主要分为课题组论文、谷歌学术、微信公众号、其他学术搜索平台

      # 6.1 课题组论文

      1. 直接从课题组的发表论文中开始看是迅速了解一个课题组的研究方向的方法，如果老板被约稿写过相关综述文章最好看看那篇综述；

      2. 也可以从导师的论文中，看看他引用的综述或者高引用量的文献，这些也是可以帮助了解课题组研究内容的文献。

      # 6.2 谷歌学术

      1. 可以在谷歌学术上查找到自己导师发表的论文；

      2. 谷歌学术的搜索功能挺全的，也和EndNote有比较好的联动，可以自行学习。

      # 6.3 微信公众号

      这是一个比较容易被忽视的地方，但是其实一些比较好的公众号会分享一些领域内比较好的文章，甚至会附带一些总结和讨论，是刷手机时获取信息的一个优秀选择。

      # 6.4 其他学术搜索平台

      现有的学术搜索平台还是挺多的，分别是新兴平台，经典平台和专有平台。

      新兴平台：

      比如我使用过清华开发的Aminer平台，半年前还是一个免费平台，半年后就开始收费，并且用户界面也经常改动，但是有点是当查找一篇论文时，会自动递归出相关索引论文，再根据引用数绘制热点图。（但是读多了也会发现这个功能其实比较一般）

      经典平台：

      SCI-HUB，Sciencedirect

      专有平台：

      针对每个领域，可能会有一个仅仅针对这一领域的文献搜索平台，可以去多多了解。（比如一些领域顶刊的官网）

      # 7. 彩蛋

      根据某位清华电子系的同学的介绍，初级阶段，阅读论文的数量和等级大概如下分布：

      20篇及以下，略有耳闻；

      21-50篇，认知模糊；

      51-100篇，拜入门下；

      101-250篇，初窥门径；

      251-500篇，领域入门。

      根据张老师的指导，在一个领域工作3000小时算做入门来看，平均1篇论文看2小时，那大约1/3的时间都在进行论文阅读，所以对于科研文献的初步认知，科学管理，目的性阅读都是研究生初期阶段需要认真培养的技能点。
