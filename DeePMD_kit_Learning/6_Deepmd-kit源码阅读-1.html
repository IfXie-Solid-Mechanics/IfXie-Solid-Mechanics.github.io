    # Deepmd-kit源码阅读

    # 1. 阅读软件的结构

    一般计划深度地学习一个软件时，当然可以直接找到src然后阅读源码，但是我的个人习惯是先了解软件的文件分布以及各个分布的意义后，再阅读细节的代码部分

    对于Deepmd-kit，其主要目录结构如下（根据GitHub的readme初步推测）：

    1. deepmd目录，Python模态库: 存放着DeePMD-kit python modules。

    2. source/lib目录，核心库：这是项目的基石，包含了基础数据结构、算法和其他必要的函数。这些代码为整个框架提供了基本功能，如数值计算和文件处理，确保其他模块能够高效运行。

    3. source/op目录，操作符实现: 这里的OP可能指的是运算节点或特定功能模块，类似于机器学习框架中的运算单元。它们负责执行具体的算法任务，支撑模型的训练和推理过程。

    4. source/api_cc目录: C++ API: 为开发者提供使用C++调用框架功能的能力，便于集成到其他C++项目中，扩大了框架的应用范围。

    5. source/api_c目录: C API: 为开发者提供使用C语言调用框架功能的能力，便于集成到其他C语言项目中，扩大了框架的应用范围。

    6. source/nodejs目录: Node.js API：支持JavaScript开发者在服务端环境中使用框架功能，可能用于数据处理或作为网络服务提供 API接口。

    7. source/ipi目录:i-PI客户端：实现与外部系统（如分布式计算平台）的连接，允许与其他工具集成，扩展框架的应用场景。

    8. source/lmp目录:LAMMPS模块：将将框架功能嵌入到LAMMPS中，支持分子动力学模拟中的深度势能模型应用，丰富了模拟方法。

    如果是生物分子方向，Deepmd-kit也提供了在Gromacs框架下的实现，但是由于与我自己的研究方向比较远，后续不予讨论。

    # 2. 阅读source/lib目录

    基于上述部分的分析，deepmd-kit软件的核心部分是source/lib目录，所以将这一部分设置为源码阅读的起点。

    # 2.1 source/lib目录结构

    还是先分析目录的代码结构，然后再进一步地细看代码内容，在source/lib目录下，主要模块分别是：

    1. include目录，主要是一些C++的库文件，并且标注了including DPA-2 supports，这个和DPA这个LAM有关。

    2. src目录，源码

    3. tests目录，测试目录

    4. CMakelist.txt，程序编译CMake文件

    所以，先初步解读CMakelist，然后阅读src目录，最后再回头分析CMakelist，中间穿插看看其他部分是这一阶段的目标；

    最后从不知道的CMAKE_CURRENT_BINARY_DIR这个位置，引用了一个版本库文件，之后可以稍微留意。

    然后是一个函数定义：function(create_ library _suffix)

    这个函数的作用是创建一个共享库（shared library）。函数参数是一个后缀字符串_suffix。以下是函数的内部操作：

    • set(libname_suffix "${libname}${_suffix}")：将传入的_suffix附加到libname后面，生成新的库名字。
    • add_library(${libname_suffix} SHARED ${LIB_SRC})：根据收集来的源文件创建一个共享库，名字是刚刚生成的 libname_suffix。
    • 配置包含目录：通过target_include_directories设置公共的头文件路径，包括当前源代码的include目录和安装后 的include目录。

    总而言之，就是把scr中的源码设置了一个公共的头文件路径。

    1. 软件编译所需要的源码文件来源于src，并且下一步工作就是阅读其中主要的cpp源码；
