      # LAMMPS在使用DP势函数时的Debug

      # 1. Deepmd势函数的元素序号

      Deepmd的势函数是有默认元素排序的

      我的的训练集的map是(Co-0, Ni-1, Pd-2, Cr-3, Fe-4)，这一步是之前使用dpdata插件得到的

      我的的势函数训练的input.json中的map是(Cr-0, Fe-1, Co-2, Ni-3, Pd-4)，这一步是我自己设置的，理论上应该相互匹配，但是之前的工作中没有注意这一点。

      # 2. LAMMPS中的元素序号

      在我初步对于Ni和Pd进行单轴拉伸时，两者的序号都是1，而且结果几乎相同，而且与Ni的密度泛函计算结果接近。

      因此我认为原子序号的不匹配可能是我这一错误的最大原因，因此我先尝试将Pd的需要修改为2，然后再进行一次测试。

      # 2.1 修改原子序号代码

      使用文本编辑器打开Pd_32000.lmp文件，做以下修改：

             32000  atoms
           2  atom types
      1
      2
      Masses

            1   58.69000000             # Ni
            2   106.42000000            # Pd
      1
      2
      3
      4

      这里通过lammps进行原子序号的修改:

      units           metal
boundary        p p p
atom_style      atomic
timestep        0.001
neighbor        0.2 bin
read_data       Pd_32000.lmp
set             type 1 type/ratio 2 1.0 8793
write_data      Pd_32000_2.lmp

      2
      3
      4
      5
      6
      7
      8

      运行命令

      lmp -i in.exchange
nohup lmp -i in.test > output.log 2>&1 &

      # 3. 修改Pd原子序号后的实验结果分析

      # 3.1 初步修改结果

      好的地方：的确在结果方面得到了不同；

      坏的地方：结构优化后的结果是bcc结构，由于bcc结构没有做过应力应变曲线，所以最终得到的应力应变曲线也比较抽象。

      结构优化后的构型图片：

      测试应力应变曲线：

      此外，元素1对应一个基态为fcc的构型或者难以从fcc相变为其他相的元素，元素2对应一个基态为bcc的构型，那么我现在更愿意相信最终的结果是(Cr-1, Fe-2, Co-3, Ni-4, Pd-5)

      # 3.2 Deepmd-kit官网查询

      我再次翻阅了Deepmd-kit的相关描述，发现在某个不起眼的位置，进行了下述说明：

      总而言之，根据type_map的设置与否这个前提分了两种情况（明显能够看出来还是设置一下比较好）

      在设置type_map后，势函数应该就是和type_map一致，而且默认序号也是从1-5

      如果希望调整默认序号，则需要从pair_coeff后面自己修改，保证pair_coeff后面跟着的修改元素顺序和lammps设置的顺序相同即可。

      也就是之前算的和Ni比较接近的结果可能用的是Cr元素的势函数，并且可能Cr元素在300K下的fcc相比较难相变为bcc相，而Fe元素就比较容易。
