    # VASP生成DeePMD-kit训练数据的一种更高效的思路

    相比于高精度地计算AIMD，然后直接使用AIMD的结果作为dataset是一个比较低效的方法，需要一种更加高效的方法来提升工作流。

    方法的核心思想是将AIMD与Structure optimization结合，其中AIMD用于高效地探索相空间，然后使用Structure optimization详细地计算力与能量。

    # 1 一种效率极高的AIMD计算

    效率提升部分：

    1. 将K点设置为1，1，1从而提高积分速度，预计效率提升在7倍左右；

    2. 将赝势从pv赝势调整为普通赝势，每原子价电子数目下降6个，总电子数下降至原来的2/3左右，预计效率大幅提升；

    3. 将ALGO改为VeryFast提高电子步的收敛速度，预计效率将小幅提升；

    4. 将程序从vasp_std转为vasp_gam可以将运行效率提升为原来的1.5倍。

    综上提升，整体效率提升约为85倍，运行1600离子步约花费2小时。

    # 2 从AIMD中提取构型

    # 2.1 两种读取方式说明：

    从AIMD中提取POSCAR文件有两种思路，第一种思路是在AIMD运行的过程中，从CONTCAR文件中读取，第二种思路是在AIMD运行完成后，从XDATCAR文件中读取。

    第一种思路：难在监控AIMD运行，由于CONTCAR文件是AIMD运行时，当前构型的离子实位置信息，因此需要监控AIMD运行过程；

    第二种思路：在AIMD运行完成后，一下子把所有构型文件从XDATCAR文件中提取出来即可。

    在我的理解中，从XDATCAR文件中读取构型信息是比较容易实现的，因此本文也会继续详细介绍从XDATCAR文件中读取的方法。

    # 2.2 从XDATCAR文件中读取POSCAR文件的方法

    首先先展示基础bash脚本如下：

    #!/bin/bash
sum_atoms=$(cat XDATCAR | sed -n '7p' | awk '{for (i=1;i<=NF;i++) sum+=$i; print sum}')
sum_steps=$(grep F= OSZICAR | awk '{print $1}')
for j in $sum_steps
do
awk '{if (NR<8) print $0}' XDATCAR > POSCAR-$j
grep -xA $sum_atoms "Direct configuration=  $j" XDATCAR >> POSCAR-$j
grep -xA $sum_atoms "Direct configuration=   $j" XDATCAR >> POSCAR-$j
grep -xA $sum_atoms "Direct configuration=    $j" XDATCAR >> POSCAR-$j
grep -xA $sum_atoms "Direct configuration=     $j" XDATCAR >> POSCAR-$j
done

    用awk读取出盒子信息；

    说明一下grep的特殊写法，是根据本人在2024年对于XDATCAR文件读取时，发现如果离子步对应的是个位数，在XDATCAR文件中Direct configuration=后面是5个空格；如果离子步对应的是两位数，在Direct configuration=后面是4个空格；如果离子步对应的是三位数，在Direct configuration=后面是3个空格。至于四位数及以上的AIMD我没跑过，不了解，但是猜测四位数的Direct configuration=后面是2个空格（待更新，跑完4位NSW的AIMD就知道了）。

    由于我是在每40个离子步的间隔取一个构型，所以对于bash脚本进行进一步地修改如下

    #!/bin/bash
sum_atoms=$(cat XDATCAR | sed -n '7p' | awk '{for (i=1;i<=NF;i++) sum+=$i; print sum}')
sum_steps=$(grep F= OSZICAR | awk '{print $1}')
for j in $sum_steps
do
    # 判断 j-1 是否是 40 的倍数或 0
    if [ $((j - 1)) -eq 0 ] || [ $(( (j - 1) % 40 )) -eq 0 ]; then
        awk '{if (NR<8) print $0}' XDATCAR > POSCAR-$j
        grep -xA $sum_atoms "Direct configuration=  $j" XDATCAR >> POSCAR-$j
        grep -xA $sum_atoms "Direct configuration=   $j" XDATCAR >> POSCAR-$j
        grep -xA $sum_atoms "Direct configuration=    $j" XDATCAR >> POSCAR-$j
        grep -xA $sum_atoms "Direct configuration=     $j" XDATCAR >> POSCAR-$j
    fi
done

    # 2.3 运用python与ASE库读取POSCAR文件

    首先先简单地学习西安服务器上的extract_config_npt.py脚本

    #!/usr/bin/env python
"""
Function: extract configurations of some interval steps from AIMD(nvt/npt); Input files: XDATCAR; Output files: POSCAR-*
Usage: python3 extract_config_v2.py
"""

## Import relevant library
from ase.io import read, write

## Input parameters
start = int(input('starting step:\n'))
interval = int(input('interval of steps:\n'))
nblock = int(input('NBLOCK:\n'))

## Read all trajectory from OUTCAR
traj = read('XDATCAR', index='::1', format='vasp-xdatcar')  ## read AIMD trajectory, interval is 1;
print('Read all {} trajectory structures'.format(len(traj)))

## Write interval configurations in POSCAR
num_extract = 0
for i in range(int(start/nblock)-1, len(traj), int(interval/nblock)):
    num_extract += 1
    write('POSCAR-{:03d}'.format(num_extract), traj[i], format='vasp', direct=True)
print('Extract {} trajectory structures'.format(num_extract))

    我对于原脚本的write部分进行了修改，因为在使用ase-3.23.0时，共享文件夹中的脚本含有一个bug，即多写了一个label变量，python运行后会报错如下

    1

    因此删除原脚本中的label变量即可，并且可以保持POSCAR-{:03d}，因为是用num_extract来定义POSCAR标签，所以如果抽样的样本小于1000都是可以使用POSCAR-{:03d}。如果抽样的样本大于或等于1000则可以使用POSCAR-{:04d}，但是这种情况目前感觉十分少见。

    至于ase库在scnet或者超算上安装，需要找超算技术人员帮忙安装一个conda环境，在里面配置ase库，因为ase库的依赖比较复杂，有的库如果没有对应环境，学生在超算上是无法下载对应环境的。

POTCAR_PATH="$WORK_DIR/POTCAR"
KPOINTS_PATH="$WORK_DIR/KPOINTS"

# List of job directories
JOB_DIRS=$(ls POSCAR-*)

for JOB_DIR in $JOB_DIRS; do
    # Go into the job directory
    cd $WORK_DIR
    DIR_NAME="dir_$JOB_DIR"

    cd $WORK_DIR/$DIR_NAME

    # Create symbolic links to the INCAR and POTCAR files
    ln -s $INCAR_PATH INCAR
    ln -s $POTCAR_PATH POTCAR
    ln -s $KPOINTS_PATH KPOINTS
    ln -s $WORK_DIR/$JOB_DIR POSCAR
    
    # Copy the Slurm script template into the job directory
    cp $WORK_DIR/$SLURM_TEMPLATE .

    # Submit the job
    sbatch $SLURM_TEMPLATE
done

    # 3.3 处理VASP输出结果作为DeePMD-kit的输入文件

    这里参考张寅老师的github代码(https://github.com/yzhang951/MD-utility/blob/main/DeePMD/data_conv.py)

    import dpdata
import numpy as np

vasp_multi_systems = dpdata.MultiSystems.from_dir(
    dir_name="./", file_name="*OUTCAR", fmt="vasp/outcar"
)
training_set = dpdata.MultiSystems()
validation_set = dpdata.MultiSystems()
#print(vasp_multi_systems.systems)
#vasp_multi_systems.to_deepmd_raw("./deepmd_data/")
#vasp_multi_systems.to_deepmd_npy("./deepmd_data/")

for system in vasp_multi_systems:
    frame_num = len(system)
    index_validation = np.random.choice(frame_num, size=int(0.2*frame_num), replace=False)
    index_training = list(set(range(frame_num))-set(index_validation))

    data_training = system.sub_system(index_training)
    training_set.append(data_training)
    if len(index_validation)>0:
        data_validation = system.sub_system(index_validation)
        validation_set.append(data_validation)
#    print('# the training data contains %d frames' % len(index_training))
#    print('# the validation data contains %d frames' % len(index_validation))

print(training_set)
print(validation_set)
training_set.to_deepmd_raw('training_data')
training_set.to_deepmd_npy('training_data')
validation_set.to_deepmd_raw('validation_data')
validation_set.to_deepmd_npy('validation_data')

    # 4.1 在ZHANG01机器上完成VASP的AIMD输入文件的创建

    创建的文件有POSCAR，POTCAR，INCAR，在Fe目录下使用create.sh脚本如下：

    #!/bin/bash
for i in $(seq 1 1 12)
do
    mkdir "Fe_300K_distorted_${i}"
    cd "Fe_300K_distorted_${i}"
    atomsk --create fcc 3.450 Fe -dup 3 3 3 tmp.xyz

    # 根据 i 的不同值选择不同的 deform 操作
    if [ "$i" -eq 1 ]; then
        atomsk tmp.xyz -deform x 0.05 POSCAR
    elif [ "$i" -eq 2 ]; then
        atomsk tmp.xyz -deform x -0.05 POSCAR
    elif [ "$i" -eq 3 ]; then
        atomsk tmp.xyz -deform y 0.05 POSCAR
    elif [ "$i" -eq 4 ]; then
        atomsk tmp.xyz -deform y -0.05 POSCAR
    elif [ "$i" -eq 5 ]; then
        atomsk tmp.xyz -deform z 0.05 POSCAR
    elif [ "$i" -eq 6 ]; then
        atomsk tmp.xyz -deform z -0.05 POSCAR
    elif [ "$i" -eq 7 ]; then
        atomsk tmp.xyz -deform xy 0.05 POSCAR
    elif [ "$i" -eq 8 ]; then
        atomsk tmp.xyz -deform xy -0.05 POSCAR
    elif [ "$i" -eq 9 ]; then
        atomsk tmp.xyz -deform yz 0.05 POSCAR
    elif [ "$i" -eq 10 ]; then
        atomsk tmp.xyz -deform yz -0.05 POSCAR
    elif [ "$i" -eq 11 ]; then
        atomsk tmp.xyz -deform xz 0.05 POSCAR
    elif [ "$i" -eq 12 ]; then
        atomsk tmp.xyz -deform xz -0.05 POSCAR
    fi
    rm tmp.xyz
    cp ~/POTCAR_vasp/VASP/Fe/POTCAR .
    cp ~/2_python_code/nose_mass.py .
    
    # 捕获python脚本的输出
    SMASS=$(python3 nose_mass.py -T 300 -f 20 -u fs | grep 'SMASS' | awk '{print $3}')
    
    # 使用SMASS值替换INCAR文件中的默认SMASS值
    cat >INCAR <<!

ISPIN = 1

TEEND = 300

PREC = Normal

EDIFF = 1E-4

ALGO = VeryFast

KSPACING = 0.8
!

    cp ~/07_bash_code/xian_vasp_gam/cal.slurm .
    cd ..
done

for i in $(seq 0 1 2)

    cd "Fe_${i}300K"

    # 复制 POTCAR 和其他文件
    cp ~/POTCAR_vasp/VASP/Fe/POTCAR .
    cp ~/2_python_code/nose_mass.py .
    
    # 计算 SMASS，注意算术运算的正确使用

    cat >INCAR <<!
IBRION = 0

TEBEG = $((300 + i*1000))

ALGO = VeryFast

KSPACING = 0.8

source /work/home/if_xie/apprepo/vasp/6.4.1-ioptcell_intelmpi2017_hdf5_libxc/scripts/env.sh

export MKL_DEBUG_CPU_TYPE=5 #加速代码
export MKL_CBWR=AVX2 #使cpu默认支持avx2
export I_MPI_PIN_DOMAIN=numa #内存位置与cpu位置绑定，加速内存读取。对于内存带宽要求高的计算提速明显

# Dirrectory containing the jobs
WORK_DIR=$(pwd)

# Paths to the INCAR and POTCAR files
INCAR_PATH="$WORK_DIR/../INCAR"
POTCAR_PATH="$WORK_DIR/../POTCAR"

# List of job directories
JOB_DIRS=$(ls POSCAR-*)

for JOB_DIR in $JOB_DIRS; do
    # Go into the job directory
    cd $WORK_DIR
    DIR_NAME="dir_$JOB_DIR"
    mkdir $DIR_NAME
    cd $WORK_DIR/$DIR_NAME

    # Create symbolic links to the INCAR and POTCAR files
    ln -s $INCAR_PATH INCAR
    ln -s $POTCAR_PATH POTCAR
    ln -s $WORK_DIR/$JOB_DIR POSCAR

    srun --mpi=pmi2 vasp_std
done

    # 4.5 从Fe_result过渡到Fe_traning_data

    这一步只是提取Fe_result中计算得到的OUTCAR文件，存储到Fe_traning_data目录

    #!/bin/bash
for dir in result_*/ 
do
    echo "进入目录：$dir"
    cd "$dir"  

    for dir1 in dir_*/  
    do
        echo "进入目录：$dir1"
        cd "$dir1"

        target_dir=~/Fe_training_data/$(basename "$dir")/$(basename "$dir1")
        mkdir -p "$target_dir"  

        if [[ -f "OUTCAR" ]]; then
            cp OUTCAR "$target_dir/OUTCAR"
        else
            echo "OUTCAR 文件不存在：$dir1"
        fi

        cd .. 
    done

    cd ..  
done
