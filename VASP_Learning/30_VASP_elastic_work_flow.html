    # VASP应力应变计算弹性模量工作流

    本篇工作流介绍了整个workflow的物理合理性，但是由于应变的范围较大，计算模量时的误差也是较大的，并且没有打开磁性分析，因此对于最终模量结果的正确性计算，详见下一篇工作流。

    # 0. 前言

    基于第一性原理计算应力、应变、弹性模量本身就是一个比较交叉学科的事情。（我问过某个985物理系的研究生，其甚至不知道应力的概念）（当然，力学系的绝大部分同学也不了解第一性原理的基础概念和细节）

    所以网上的资料是比较少的，即使有资料，大概也有很多细节没有考虑到，导致出现各种各样的错误。（比如单老师的计算方法与实验误差极大，且维加德定律公式内容还打错了）（比如结构优化开ISIF=3来操作）

    而我通过自己的实践，总结了一份逻辑还算自洽，结果还算看得过去的工作流，以此文分享。

    # 1. 结构优化

    先查找结构的经验晶格常数（推荐：https://www.ctcms.nist.gov/potentials/）

    但是这些经验晶格常数仅限于用于设置最初的POSCAR构型，不建议直接跳过结构优化来进行下一步操作：

    主要原因:这些经验晶格常数的精度往往是不够的，VASP中SYMPREC的默认值都是10−5埃，这些经验晶格常数精度几乎都低于这一精度。而在VASP中，百分之一的晶格常数误差就可能导致2meV/atom的势能计算误差。

    # 1.1 结构优化中的ISIF设置

    从VASP的官方wiki中可以查看ISIF各个取之的含义；

    网络上ISIF取3的方法，不合理之处有2个：

    1. 如果目标的基态构型是bcc结构，但是您想模拟的是fcc结构，那么ISIF=3会让您的fcc结构向着bcc的结构的方向变化。（虽然存在一个能垒，使得很难完全就相变过去了，但是这种趋势的影响不能忽略）

    2. 如果你研究的构型基态就是fcc结构，那么开ISIF=3可能会影响正交性，甚至影响x轴指向稍稍偏离原本想计算的[100]发现，那么结果潜在地被作用了一个坐标变化，引入了误差。

    而VASP更新的ISIF取8，在我们这个工作流中是一种不错误，但是及其不经济的行为：

    1. 不错误的原因是其的确规避了上述两个问题；

    2. 不经济的原因是，一方面在后续的工作流中需要进行ISIF=2来使得变形更加符合Cauchy-Born rule，所以离子实位置优化是会被操作的，但是不应该在这一步；另一方面，这里由于体积变化导致的Pulay stress引入的误差，导致计算出来的能量几乎是不可信的，存在1-4meV/atom的误差，所以费力地算这个也是有些亏了。

    综上所属，建议在结构优化时，ISIF取7。

    # 1.2 结构优化中的ENCUT

    理论上取正无穷肯定是最好的

    推荐取ENCUT=1.3×max(ENMAX)，这算是VASP推荐的精度和算力的一个平衡点。

    这里主要是应对Pulay stress的影响。

    # 1.3 KSPACING的设置

    由于提高了ENCUT，这里可以选择对应的提高KSPACING来进一步地提升计算精度。

    推荐取KSPACING=0.1

    同样也是应对Pulay stress的影响。

    # 1.4 re-relax操作

    由于Pulay stress的影响，在您进行上述两个操作之后，可能还存在优化不到平衡位置的问题（比如Cr元素的FCC构型）

    那么按照VASP官方的建议，可以对于已经relax的构型，再进行结构优化，直到结构优化只剩1个离子步时可以停止。

    步数的话，推荐使用OSZICAR查看，一般不超过3步，OSZICAR记录内容少，一页就能看完。

    # 2. 拉伸操作

    拉伸操作这一步，默认x,y,z方向分别对应[100],[010],[001]

    其实不是也可以，以您要计算的那个晶面为准。

    # 2.1 拉伸脚本

    推荐使用atomsk来进行操作，操作脚本如下所示

    #!/bin/bash

cp POSCAR POSCAR-01
for i in $(seq 1 1 100);
do
    #le:less than or equal
    if [ $i -le 50 ]; then
            ii=$(echo "scale=6; $i/1000" | bc)
    else
            tmp=$(($i - 50))
            ii=$(echo "scale=6; $tmp/-1000" | bc)
    fi
    mkdir Pd-$(printf "%03d" $i)
    cd Pd-$(printf "%03d" $i)
    cp ../POSCAR-01 .
    atomsk POSCAR-01 -sort species up -frac -deform x $ii POSCAR
    rm POSCAR-01
    cd ..
done

rm POSCAR-01

    # 2.2 拉伸分析

    从拉伸脚本不难看出，拉伸时，控制xx分量存在应变，其余5个应变分量全部控制为0

    所以对于立方对称模型，通过C11=σx/ϵx和C12=σy/ϵx即可算出两个弹性模量。

    如果还需要计算独立的C44那么需要再类似地操作一个剪切变形即可。

    注意，这和传统的单轴拉伸控制侧面应力为0的边界条件有极大区别。所以不要直接拿单轴拉伸的应力应变曲线来验证结果。

    # 3. 应力计算步骤

    将上述的构型进行整理，即可进行这一步操作，进行应力的计算，最终目标是计算出对应模量。

    # 3.1 使得变形更加符合Cauchy-Born rule

    从变形脚本不难看出，直接对所有原子作用一个位移梯度显然是不符合Cauchy-Born rule的要求。因此，在这一步提出通过ISIF=2来微调离子坐标，保证VASP计算准确性的同时，使得计算结果更加符合Cauchy-Born rule

    并且这样也使得工作流的设计前后自洽。

    import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.metrics import r2_score

# 读取SE.dat文件，假设文件的第一行是标题
data = pd.read_csv('SE.dat', delim_whitespace=True, header=0)

x = data['Strain'].values
y = data['Energy'].values

# 进行二次多项式拟合
coefficients = np.polyfit(x, y, 2)
poly_fit = np.poly1d(coefficients)

# 计算拟合后的y值
y_fitted = poly_fit(x)

# 输出拟合得到的二次方程系数a、b、c
print(f"Quadratic equation: y = {coefficients[0]:.4f}x^2 + {coefficients[1]:.4f}x + {coefficients[2]:.4f}")

# 计算R平方值以评估拟合效果
r_squared = r2_score(y, y_fitted)
print(f"R-squared value: {r_squared:.4f}")

# 绘制图形
plt.plot(x, y, 'o', label='Data points')
plt.plot(x, y_fitted, '-', label='Quadratic fit')

# 添加标签和图例
plt.xlabel('Strain')
plt.ylabel('Energy(eV)')
plt.legend()

plt.savefig("energy_strain_fit.png")   # 将图像保存为 PNG 文件

    # 4.3 弹性模量计算脚本

    采用线性回归模型

    import numpy as np
import pandas as pd
from sklearn.linear_model import LinearRegression

# 读取SS.dat文件
data = pd.read_csv('SS.dat', delim_whitespace=True, header=0)

# 提取应变和应力数据
strain = data['Strain'].values
stress = data['Stress_Gpa'].values

# 将应变转换为二维数组以适应线性回归模型
X = strain.reshape(-1, 1)
y = stress

# 创建并拟合线性回归模型
model = LinearRegression()
model.fit(X, y)

# 获取斜率和R平方值
slope = model.coef_[0]
r_squared = model.score(X, y)

# 输出结果，保留六位小数显示斜率和四位小数显示R²
print(f'斜率为: {slope:.6f}')
print(f'R平方值为: {r_squared:.4f}')
