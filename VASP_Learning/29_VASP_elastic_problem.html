# VASP问题集整理-20250309

# 1. 对于同一个构型有两种计算结果

# 1.1 ISIF=7结构优化的计算结果

我们希望通过VASP执行ISIF=7的结构优化计算得到relaxed structure:

在结构优化的最后一步，我们可以得到能量结果，即OSZICAR文件的最后一行（单位为电子伏特）：

$ F = -.56418508E+03 $

# 1.2 ISIF=2结构优化的计算结果

ISIF=2由于不改变晶胞体积，而且更加符合Cauchy-Born rule，因此这一计算结果更加准确，通过这一个计算来计算出我们应变后构型的能量。

在应变的第0个构型，即还没有应变的构型，即直接从ISIF=7的结构优化中得到的最终CONTCAR文件，直接复制过来的POSCAR文件，最终能力计算结果为

$ F = -.56407055E+03 $

# 1.3 两种计算结果更为直观的差别体现

如果我直接给两个数字，也许您会认为这两者之间并无差别，但是如果我直接给出它们的能量在能量-应变曲线上的结果，也许您就能更加直观的感受到两者的差别：（第一张图是ISIF=7的计算结果，第二张图是ISIF=2的计算结果）

# 1.4 造成这一能量误差的原因

首先，我现在还不足以判断造成误差的原因。只有几个零碎的思路：

  1. 对于ISIF=7计算收敛的结构再次用ISIF=7重新计算结构收敛，看看能量最终结果是否改变，看看构型最终结果是否改变；

  2. 对于ISIF=2改变kpoint的数量再次计算构型周围应变的情况，因为直接遇到过同样能量阶跃问题，是通过调整kpoint导致阶跃消失。（我也试过，调整ENCUT仍然存在outlier现象，请您移步上一篇网页进行了解）

# 2. 对于同一个构型的三种计算结果

# 2.1 ISIF=2更密k网格的结构优化

能量结果$ F = -.56371722E+03 eV$如果和之前的结果比较，这里的误差达到了4meV/atom

# 3. 在k点更加细密的情况下进行ISIF=7的结构优化

如果您将之前的k点正常的ISIF=7的优化结果，带入k点加密的ISIF=7的结构优化中进行处理，理论上可以得到一个更加准确的relaxed构型

在2025年3月9日晚，我尝试这样操作了

# 3.1 ISIF=7更密k网格的结构优化

能量结果为$ F = -.56371775E+03 eV$，不难发现，这个结果和2.1中的结果几乎没有差别

# 3.2 ISIF=2更密k网格的结构优化

将3.1结构优化得到的CONTCAR，您显然可以进行ISIF=2的结构优化，鉴于1.1和1.2的分析，您可以猜测到两者应该是存在些许不同的。

而在您的ISIF=2更密k网格的结构优化计算结果中，$ F = -.56372831E+03 eV$

首先，误差在k点加密后的确得到了一个数量级的减小；

其次，其次误差范围显然还是大于您设置的EDIFF=1E-4这一量级。
