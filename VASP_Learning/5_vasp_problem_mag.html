      # VASP中对于FCC晶体磁矩计算的问题（与ISMEAR相关）

      # 1 影响磁矩计算的变量：

      # 1.1 元素的种类

      不同元素有区别，这是平凡的结论；

      # 1.2 晶胞的大小

      结论1:在其他影响magnet的值保持一定时，仅含1个原子的原胞，相比于3×3×3的超晶胞，两者的平均磁矩不同；

      # 1.3 晶格常数的大小

      结论1:在其他影响magnet的值保持一定时，无论是仅含1个原子的原胞，还是3×3×3的超晶胞，改变晶格常数也会对计算结果产生不同程度的影响。

      # 1.4 ENCUT的大小

      结论1:在其他影响magnet的值保持一定时，无论是仅含1个原子的原胞，还是3×3×3的超晶胞，改变ENCUT的值也会对计算结果产生较小的影响。

      # 1.5 SIGMA的大小（ISMEAR的参数）

      结论1:在其他影响magnet的值保持一定时，无论是仅含1个原子的原胞，还是3×3×3的超晶胞，改变SIGMA的值也会对计算结果产生较小的影响。

      # 1.6 KSPACING的大小

      结论1:在其他影响magnet的值保持一定时，对于仅含1个原子的原胞，改变KSPACING不影响magnet的计算结果；

      结论2:在其他影响magnet的值保持一定时，对于3×3×3的超晶胞，改变KSPACING明显影响magnet的计算结果；

      补充1：上述结论中，“改变KSPACING”不仅指改变数值，而且也改变了在倒空间的k点个数；

      结论3:对于3×3×3的超晶胞，如果在不同的KSPACING下，magnet随SIGMA的单调性发生改变。

      # 2 对于上述结论的问题思考

      # 2.1 可以解决的问题

      对于1.1,1.2都是可以在计算时进行规避的问题，即仅计算各个元素的超晶胞；

      # 2.2 暂时妥协的问题

      对于ENCUT，其影响对于各个物理参数都是较小的，但是时间成本却上升得很快，可以被放在最后一步来调整，做时间-精度平衡；

      对于晶格常数的选取，目前是先参考Mishin的结论用于Fe以及Ni上，后续弄清后再计算剩下3种晶体；

      Fe,fcc,3.483埃,1.342 μB/atom

      Ni,fcc,3.519埃,0.631 μB/atom

      # 2.3 无法妥协或难以解决的问题

      SIGMA和KSPACING的影响是复杂的，不但是从理论上去分析就较为复杂，即使是单纯地描述这个问题就很复杂；

      无法妥协的原因：在AIMD中同样需要设置这两个参数，并且找不到Mishin那样的“金标准”；

      难以解决的原因：

      KSPACING物理意义是倒格子，理论上其越细密，在布里渊区积分结果越接近真实值。（我验证了具体撒点个数和KSPACING值的关系）

      但是magnet不是布里渊区积分结果，而是一个迭代量。（这仅是我个人了解VASP流程的认知）

      但是我测试Fe的FCC超晶胞的KSPACING=0.35和KSPACING=0.25的迭代电子步仅相差2步。

      # 3 对于上述结论的算例支撑

      对于晶格常数为3.483埃的Fe超晶胞，取KSPACING=0.25时,每个方向3个格点；

      对于晶格常数为3.483埃的Fe超晶胞，取KSPACING=0.25时,每个方向2个格点；

      # 4 对于ISMEAR的补充

      在VASP计算中，ISMEAR如果大于0，则仅能取正整数N，表示Methfessel Paxton Smearing的阶数，具体理论如下；

      对于δ(x)的N阶近似，δ(x)N=∑n=0NAnH2ne−x2

      由于Hermite多项式在作用e−x2后正交归一化系数为n!2nπ

      所以取An=H2n(0)(2n)!4nπ=(−1)nn!4nπ

      所以取x=ϵ−μσ即为这一方法的估计。

      很明显，如果取0阶时，即为高斯正态分布。

      而Methfessel & Paxton对于这一数学操作应用于费米面后，在计算速度上的优化与在误差上的缺陷都明确地给出了更加物理的分析。

      误差上引入的“电子熵”如果小于0.001eV/atom都是在可以接受的范围以内。

      可以通过下面命令查看，但是这是总熵乘以温度，还需要除以原子数来和0.001对比。

      grep "entropy T" OUTCAR
