            # 一元金属VASP计算工作流

            # 1 INCAR文件

            对于INCAR文件，需要修改的参数有NSW,SMASS,TEBEG,TEEND,KSPACING

            NSW修改说明：有的构型需要40个snapshots，对应NSW=1600，有的构型只需要20个snapshots，对应NSW=800。（每40fs取一个snapshot）

            TEBEG，TEEND修改说明：在一元，三元，五元合金问题，温度需要设置3个等级：300K（室温），1300K（低于熔点温度），2300K（高于熔点温度）

            SMASS修改说明：在修改POSCAR或者温度时，需要重新计算；

            KSPACING说明：目前是均使用3，3，3网格，后期可能修改为5，5，5网格来添加更加准确的数据，可能修改。

            # 2 一元金属POSCAR文件

            对于一元金属POSCAR文件，在确定基态的晶格常数后便可以给出基态构型，然后用atomsk建模应变构型。

            # 3 一元金属POTCAR文件

            对于一元金属POTCAT文件均使用对应元素的POTCAR文件即可；

            # 4 一元金属的工作流

            对于非应变部分，由于仅有18种构型，数量较少，以及想确定超算是否能够正常运行，因此都是手搓的；

            下面对于应变的72种构型进行详细地说明：

            生成铁的12种构型脚本create_Fe.sh如下,在ZHANG01的ifxie目录下运行:

            #!/bin/bash
for i in $(seq 1 1 12)
do
    mkdir "${i}_Fe_300K_distorted"
    cd "${i}_Fe_300K_distorted"
    atomsk --create fcc 3.450 Fe -dup 3 3 3 tmp.xyz

    # 根据 i 的不同值选择不同的 deform 操作
    if [ "$i" -eq 1 ]; then
        atomsk tmp.xyz -deform x 0.05 POSCAR
    elif [ "$i" -eq 2 ]; then
        atomsk tmp.xyz -deform x -0.05 POSCAR
    elif [ "$i" -eq 3 ]; then
        atomsk tmp.xyz -deform y 0.05 POSCAR
    elif [ "$i" -eq 4 ]; then
        atomsk tmp.xyz -deform y -0.05 POSCAR
    elif [ "$i" -eq 5 ]; then
        atomsk tmp.xyz -deform z 0.05 POSCAR
    elif [ "$i" -eq 6 ]; then
        atomsk tmp.xyz -deform z -0.05 POSCAR
    elif [ "$i" -eq 7 ]; then
        atomsk tmp.xyz -deform xy 0.05 POSCAR
    elif [ "$i" -eq 8 ]; then
        atomsk tmp.xyz -deform xy -0.05 POSCAR
    elif [ "$i" -eq 9 ]; then
        atomsk tmp.xyz -deform yz 0.05 POSCAR
    elif [ "$i" -eq 10 ]; then
        atomsk tmp.xyz -deform yz -0.05 POSCAR
    elif [ "$i" -eq 11 ]; then
        atomsk tmp.xyz -deform xz 0.05 POSCAR
    elif [ "$i" -eq 12 ]; then
        atomsk tmp.xyz -deform xz -0.05 POSCAR
    fi
    rm tmp.xyz
    cp ~/POTCAR_vasp/VASP/Fe_pv/POTCAR .
    cp ~/2_python_code/nose_mass.py .
    
    # 捕获python脚本的输出
    SMASS=$(python3 nose_mass.py -T 300 -f 20 -u fs | grep 'SMASS' | awk '{print $3}')
    
    # 使用SMASS值替换INCAR文件中的默认SMASS值
    cat >INCAR <<!
IBRION = 0

NPAR = 8

ISMEAR = 1
SIGMA = 0.2

ISPIN = 1

ISYM = 0

NSW = 1600
POTIM = 1.0

MDALGO = 2
SMASS = $SMASS

TEBEG = 300
TEEND = 300

PREC = Normal

EDIFF = 1E-4

ALGO = Fast

KSPACING = 0.24
ENCUT = 367.986
!
    rm nose_mass.py
    cp ~/07_bash_code/cal.slurm .
    cd ..
done

            可以修改部分：

            1. 第4、5行对应的文件名；

            2. 第6行的元素种类和晶格常数，扩胞的话暂时都统一是3，3，3超晶胞；

            3. 第35行对于赝势的引用（尤其是多元素时需要对这里再打个补丁）；

            4. 第39行对应的温度设置，热浴时间也可以修改；

            5. 第52行对应的总AIMD运行时长；

            6. 第58、59行对应的模拟温度；

            7. 第67行对应的KSPACING大小，已经计算过k取0.24对于所有应变都是3，3，3网格；

            8. 涉及文件路径的所有位置最好都检查一遍。(尤其是西安和乌镇服务器对应的cal.slurm脚本是有区别的，乌镇的队列叫做wzhcnormal，西安的队列叫做xahcnormal)

            # 服务器分配

            乌镇服务器：3种温度6种元素共18种基态构型+Fe与Ni在300K下的24种应变构型，共42种构型，每种构型占用64个CPU，占用总CPU数为2688；

            西安服务器：Cr，Co，Pd，Mn在300K下的48种应变构型，每种构型占用64个CPU，占用总CPU数为3072。

            估计预算：28800元。

            记住，在2024年scnet的核时是0.025元！

            服务器运行命令脚本

            #!/bin/bash
for i in $(seq 1 1 12)
do
    cd "${i}_Mn_300K_distorted"
    sbatch cal.slurm
    cd ..
done
