    # 张寅课题组服务器管理员手册第二章：磁盘管理

    # 1. 两个常用的查看命令

    # 1.1 df（英文全称：disk free）：列出文件系统的整体磁盘使用量

    1.一般可以加上选项-h（以人类可读的方式显示输出结果）

    2.一般会遇到的文件系统对象：

    tmpfs 基于内存的临时文件系统，挂在在内存上，关机时会自动被清除，但是本课题组服务器内存较大，不需频繁关机刷新内存；

    /dev/nvme0n1p1 其中，nvme0n1是一个SSD硬盘（固态硬盘）的存储器，/dev/nvme0n1p1是nvme0n1下的一个分区。/dev/nvme0n1p2、/dev/nvme0n1p3同理为第二个、第三个分区；

    【说明：组员需要知道，nvme0n1是目前本课题组服务器性能最优的磁盘，也是最贵的，推荐一般在这个目录下跑程序。管理员应该把这个硬盘挂在根目录下，使得系统程序和home目录程序都默认在这个磁盘下运行】

    /dev/sda其中，sda可能指代SCSI接口或SATA接口，而SATA接口可能接机械硬盘或固态硬盘，所以其实管理员在没有拆开服务器的前提下，无法得知是装了固态硬盘还是机械硬盘。不过从张老师那里了解到服务器5万左右的价格，保守估计是机械硬盘，毕竟一个好用的8TB的固态硬盘在2024年是7000元左右

    【说明：组员需要知道，无论sda是固态或机械硬盘，读写速度都小于nvme，所以一般推荐不在这个硬盘运行程序，而是在这个硬盘里存储文件。管理员应该把这个硬盘挂在一个特定目录下，我们目前挂载于/mnt/disk1，以后扩充硬盘的话继续挂/mnt/disk2即可】

    efivars 防火墙相关文件系统。

    3.显示挂载目录：

    df命令输出的最后一列为磁盘的挂载目录，文件存储在对应目录即为存储在对应磁盘。

    # 1.2 du（英文全称：disk used）：检查磁盘空间使用量

    与df命令不同的是du命令是对特定文件和目录磁盘使用的空间的查看，即df看系统整体，du看局部目录。

    # 2.磁盘分区

    目前，本课题组磁盘分区的意义不是很大，因为“使用功能”上，已经区分了用nvme跑程序，用sda存数据。而如果从“用户区分”的角度来看，其实可以从文件权限上进行限制。最多就是目前，对于nvme分2个区，一部分用于跑系统程序，另一部分用于跑用户程序其实意义也不大。
