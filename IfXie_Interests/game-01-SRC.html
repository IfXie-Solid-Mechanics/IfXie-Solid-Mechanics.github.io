    # StarRailCopilot

    特别鸣谢：https://github.com/LmeSzinc/StarRailCopilot (opens new window)

    SRC(StarRailCopilot) is a Star Rail auto script based on Alas(AzurLaneAutoScript).

    # 1. Install

    # 1.1 clone it from github.

    git clone http://github.com/LmeSzinc/StarRailCopilot
    1
    cd StarRailCopilot
    1

    # 1.2 set a virtual environment.

    conda create -n StarRailCopilot python==3.10.10
conda activate StarRailCopilot
    1
    2

    if the Linux or macos system are used.

    pip install -r requirements-in.txt
    1

    # 1.3 install ADB & Git

    1. ADB (Android Debug Bridge) 是一个命令行工具，可以帮助你与 Android 设备进行交互。
    brew install android-platform-tools
    1

    安装完之后在.bash-profile中export一下。

    1. Git 是一个版本控制工具，用于管理代码版本和协作开发。本人很早就开始使用git，因此已经安装了git，在IfXie-Talk中可以找到git相关教程。

    # 1.4 set up gui

    python gui.py

    2

    # 2. 使用介绍

    # 2.1 安装模拟器

    安装一款手机模拟器，并在上面安装崩坏星穹铁道或云崩坏星穹铁道。都用脚本了，肯定不挂前台，mac或Linux也挂不了前台

    对于mac用户，推荐使用蓝叠模拟器，可以直接在官网免费下载，并且包体大小算是模拟器中较小的了，整个下载安装过程在3分钟左右。

    模拟器设置推荐：平板模式，1280x720，打开ADB，机型设置为HUAWEI nova 10（有点赛博ntr的一集，主要是mhy会卡低端机型，所以选一个高端一点的安卓系统平板或手机即可）

    云星穹铁道的劣势在于脚本每次会自动登出避免占用免费时长，但是这样重新登录时可能会遇到排队现象。所以我自己下载的是星穹铁道，劣势在于占用存储空间较大。

    # 2.2 设置脚本

    在SCR中设置端口127.0.0.1:5555为模拟器 Serial，如果是其他模拟器自行查看说明文档；

    然后其他脚本任务都是简单设置设置即可（可以根据自己的需求，查看刷哪些副本）

    # 3. 说明与升华

    原作者是禁止商业使用的。

    只要设置得当，这个脚本可以通过计算机视觉的基础，模仿人的手动操作让我的帐号每天在某一固定的时间点完成游戏日常活跃度，剩下有趣的活动和高难副本可以留给自己去体验。一方面封号的概率较小，而且有较小封号可能性时，我的氪金欲望会大幅降低，另一方面不用每天花精力在这上面。

    如果想要了解更多原理性的东西，可以参考MaaFramework项目，熟练json代码和计算机视觉基础后就可以将这个项目套用在其他游戏上。通俗地说，这个脚本本质上就是融合了计算机视觉的按键精灵。但是十分不建议自己开发，因为如果游戏版本更新后加入了新的副本，那么至少需要重新截图，再重新训练模型，如果游戏ui大幅更新，那更是灾难级的。同时，这里也致谢StarRailCopilot项目每个版本稳定、快速的更新。

    我并不推荐用脚本进行游戏，但是这确实是我自己应对存在日活跃度机制的游戏的习惯。对我而言，游戏只是在人闲暇时让人娱乐的产品，而不是需要占用人每天的部分精力去完成每日日活跃的精力吞噬器。并且这也让我养成了刷GitHub的习惯，找到有意思的脚本尝试部署，甚至是比玩游戏本身还有成就感的事情，比如给Steamdeck写蓝牙脚本，得到一个低延迟蓝牙手柄
