      # 异地组网（蒲公英版本）

      # 1. 下载蒲公英客户端软件

      下载地址：https://pgy.oray.com/download/

      # 2. 初始组网

      注：仅网络管理员需要进行这一步操作

      首先组网需要先初始化一下，具体的操作就是用任意两台设备分别登录同一个蒲公英账号。然后就可以完成初始的组网了。

      组网之后，可以添加客户端成员。为成员选一个名字和密码，最好再将成员的手机号也加进来（这样的话成员用手机号登录之后就可以直接接入网络组）

      # 2.1 原理说明

      我理解的，局域网的定义是：两台以上终端，通过路由器、交换机等设备组成的通信网络就叫局域网。

      第一重的理解，可以把“蒲公英”这个软件理解为一个路由器，开放了3个线上LAN口。

      第二重的理解，“蒲公英”这个软件的运营商，打开了一台服务器，服务器提供VPN和P2P服务，实现了第一重理解中的功能。

      第三重的理解，“蒲公英”在安全性能方面，可以花时间了解VPN和P2P服务。理论上所有传输的数据包它的服务器都能有记录（但是运营商也不会闲着看一堆你打游戏期间互传的UDP数据包吧），至于害怕因为打开数据通路而被恶意植入或者读取流量的话，那么把防火墙设置好，仅对于你P2P的那个ip开放部分协议和端口。理论上这当然不是百分百的安全，但是如果只是为了联机打个游戏这种个人向的使用，安全性已经足够了（有的互联网游戏还会泄漏用户数据...）

      # 3. 用户加入操作

      先将个人手机号发送给网络管理员，然后等网络管理员添加好后，打开下载好的软件，用给到管理员的那个手机号注册并登录，并且管理员会给一个密码，输入那个密码就能直接登入网络组。

      这样用户就组入网络了。（这个软件比较好的地方就在于对于用户极度友好）

      # 4. 调整无法从mac端ping通windows端的问题

      这一部分属于遇到这个问题再来查看即可，很多数据包不互通都可以通过类似方法解决。

      # 4.1 相关知识储备

      系统防火墙：操作系统内置的一个安全软件或模块，它根据预设的规则（规则集），控制哪些网络流量可以进入（入站）或离开（出站）这台计算机。

      入站： 指从外部网络（比如同一蒲公英网络中的另一台电脑、互联网上的服务器）发送到你当前这台电脑的网络流量。

      ICMP： 全称 Internet Control Message Protocol (互联网控制消息协议)。它是 TCP/IP 协议族中的一个核心协议。最常见的用途就是 ping 命令使用的 Echo Request (回显请求) 和 Echo Reply (回显回复) 消息。

      入站 ICMP： 特指从外部发往本机的 ICMP 数据包。对于 Ping 来说，就是外部机器发来的 Echo Request 包。

      “已建立连接”的流量：如果服务器主动使得一个流量出站，比如Echo Request，在收到对应入站信息后，比如Echo Reply，那么这种入站流量就属于“已建立连接”的流量。如果其他用户莫名其妙地输入一个Echo Request入站本服务器，那么本服务器会将其判定为“未建立连接”的流量， 比如新的入站 ICMP 请求。

      # 4.2 对于这个现象的分析

      Mac 发起： Mac 主动向 Windows 发送一个 ICMP Echo Request (新的入站请求到 Windows)。

      Windows 防火墙处理（入站）： 这个 Echo Request 到达 Windows 的防火墙。Windows 防火墙很可能默认阻止了这种新的入站 ICMP 请求。 这是 Windows Defender 防火墙的常见默认行为（尤其在“公共网络”配置文件下）。因此，这个请求包被丢弃/拒绝。

      结果： Windows 没有收到有效的 Echo Request（或者收到了但被防火墙阻止处理），所以它不会生成或发送 Echo Reply。

      Mac 等待： Mac 发送了 Echo Request 后，一直在等待 Echo Reply。由于 Windows 根本没回应（或者回应被 Windows 自己的出站规则阻止的可能性较低，通常出站是允许的），Mac 最终超时，显示 Ping 不通。

      # 4.3 解决方法

      1.打开“高级安全 Windows Defender 防火墙”：

      按 Win + R 键打开“运行”对话框。

      输入 wf.msc 并按回车键。这将打开防火墙的高级管理界面。

      2.创建入站规则：

      在左侧窗格中，点击 入站规则。

      在右侧窗格中，点击 新建规则...。这将启动新建入站规则向导。

      3.选择规则类型：

      在“规则类型”步骤中，选择 自定义(C)，然后点击 下一步。

      4.选择程序：

      在“程序”步骤中，选择 所有程序（因为 ICMP 是协议层的，不关联特定程序），然后点击 下一步。

      5.配置协议和端口（最关键步骤）：

      在“协议和端口”步骤中：

      协议类型(T)： 从下拉菜单中选择 ICMPv4。

      点击旁边的 自定义... 按钮。

      在弹出的“自定义 ICMP 设置”对话框中：

      选择 特定 ICMP 类型(S)。

      在下方列表中，找到并勾选 回显请求（英文是 Echo Request，类型代码通常是 8）。这就是 Ping 命令发送的请求。

      点击 确定。

      回到主向导，点击 下一步。

      6.指定作用域：

      在“作用域”步骤中，通常保持默认设置（任何 IP 地址）即可，除非你只想允许特定网络（如蒲公英虚拟网段）的 Ping。如果需要限制：

      在“远程 IP 地址”部分，选择 下列 IP 地址，然后点击 添加... 输入你希望允许 Ping 的来源 IP（如你的 Mac 在蒲公英网络中的虚拟 IP 或整个蒲公英网段，例如 192.168.x.0/24）。

      点击 下一步。

      7.选择操作：

      在“操作”步骤中，确保选择 允许连接，然后点击 下一步。

      8.配置配置文件：

      在“配置文件”步骤中，勾选你的计算机当前连接的网络类型。通常需要勾选：

      域（如果你的电脑加入了公司域）

      专用（适用于家庭或工作网络，你信任的网络）

      公用（适用于公共网络，如咖啡厅。如果允许在公共网络被 Ping 存在安全风险，请谨慎勾选此项！ 建议只在专用网络测试）。

      重要： 确保勾选了你的 Windows 电脑当前实际连接的网络配置文件（可在系统设置 > 网络和 Internet > 属性 中查看当前网络类型）。如果规则应用的配置文件和当前网络不匹配，规则不会生效。

      点击 下一步。

      9.命名规则：

      在“名称”步骤中：

      给规则起一个易于识别的名字，例如 允许 ICMPv4 Ping (入站)。

      可添加描述，如 允许其他设备通过 ICMP Echo Request Ping 此计算机。

      点击 完成。

      # 4.4 补充

      如果是在ping向mac端时，发生这个问题，则也是类似的原因，需要修改mac的防火墙设置，这个设置风格更加linux，网络上教程也不少，理解原理后操作即可。
