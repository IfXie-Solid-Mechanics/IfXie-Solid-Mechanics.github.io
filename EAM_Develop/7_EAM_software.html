# EAM势程序设计思路

# 1. 草稿式分析

# 1.1 代码功能实现

  1. 构造目标函数；

  2. 针对目标函数进行参数优化。

# 1.2 IO分析

# 1.2.1 INPUT

构造目标函数需要输入信息，根据之前代码开发工作，我们将输入信息简单地区分为两类：第一类是基于DFT计算得到的具体数值（表面能、堆垛层错能、弹性常数等），第二类是DFT计算得到的构型文件（主要是CONTCAR文件）

虽然第一类数据通过OUTCAR等文件也能获取，但是写入程序中显然不是特别方便，这里倾向于设置一个INPUT.json文件让用户输入第一类数据；

第二类数据的CONTCAR文件路径也可以设置在INPUT.json文件中。

# 1.2.2 OUTPUT

结果输出是最终优化得到的参数；

过程可以输出优化迭代时的每一步骤参数的变化，目标函数的取值。

# 1.3 实现路径分析

首先针对Ni做出一个较好的程序，然后留出一定针对多元素的升级空间。

# 1.4 代码注释

首先尽力用较好的码风使得代码的可读性较强，注释更多起到一个补充说明的作用。（比如出现能量时，在代码中说明这是什么能量、谁的能量等）

各种各样的性质需要输入到这个json文件中（上面展示了lattice_constant这一性质），并且通常包含3个变量：第一个布尔变量表示这一性质是否需要用于构造目标函数；第二个浮点变量就是这一性质的具体数值，第三个变量表示上述第二类输入的文件位置。

然后通过python程序将各个性质的这三个变量读入后，依次设置list即可。

import json

def load_config(file_path):
    try:
        with open(file_path, 'r') as f:
            data = json.load(f)
        
        # 提取三个目标值
        config = data['settings']
        bool_val = config['enable_feature']
        float_val = config['threshold_value']
        string_val = config['user_message']
        
        # 创建结果列表
        result = [bool_val, float_val, string_val]
        return result
        
    except FileNotFoundError:
        print(f"错误：文件 {file_path} 不存在")

if __name__ == "__main__":
    config_data = load_config('input.json')
    if config_data:
        print("提取的数据列表：", config_data)
        print("类型验证：", 
              [type(item).__name__ for item in config_data])
